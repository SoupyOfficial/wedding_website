# Database Reference

## Overview

- **Engine:** SQLite (local dev) / Turso libSQL (production)
- **ORM:** Prisma — used only for schema management and migrations. All queries use raw SQL via `lib/db/query.ts`.
- **Schema file:** `prisma/schema.prisma`
- **TypeScript types:** `lib/db-types.ts` (auto-generated by `npm run generate:types`)

## Query layer

```
lib/db/client.ts      — Turso libSQL client factory (uses TURSO_DATABASE_URL if set, falls back to DATABASE_URL)
lib/db/query.ts       — query<T>(), queryOne<T>(), execute() wrappers
lib/db/transforms.ts  — toBool(), toBoolAll() — convert SQLite 0/1 to JS boolean
lib/db/utils.ts       — generateId(), now(), isUniqueViolation()
```

SQLite stores booleans as integers (0 / 1). Every query result that contains boolean fields must be passed through `toBool()` or `toBoolAll()` before use.

## Migrations

Migrations live in `prisma/migrations/`. Each migration is a timestamped folder with a `migration.sql` file.

```bash
# Create a new migration after editing schema.prisma
npx prisma migrate dev --name your-description

# Apply migrations to production Turso DB
npm run db:deploy
```

The `db:deploy` script (`scripts/migrate-turso.ts`) reads `TURSO_DATABASE_URL` and `TURSO_AUTH_TOKEN` and runs the pending SQL migrations.

## Regenerating TypeScript types

`lib/db-types.ts` is generated from `prisma/schema.prisma` by `scripts/generate-db-types.ts`. Run it whenever you change the schema:

```bash
npm run generate:types
```

The `prebuild` npm hook runs this automatically before every Next.js build.
